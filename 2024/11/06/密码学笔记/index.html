消息认证的必要性

安全的对话要确定消息发送方的身份和以及消息未被串改，仅加密在很多时候并不能确保这些.

MAC

给我们的消息打一个tag！

定义

消息认证码，即 MAC 是一个 PPT 算法组成的三元组 \(\Pi=(Gen,Mac,Vrfy)\): + 密钥生成算法: \(k \leftarrow Gen(1^n) , |k| \geq n\). + 标签生成算法：\(t \leftarrow Mac_k(m)\ or\ t := Mac_k(m), m\in \{0,1\}^*\). + 确定性的认证算法：\(b := Vrfy_k(m,t) , b=1 \ iff.\ m\ is\ valid\).

正确性与安全性要求

正确性：正确生成的MAC要被认可，即 \[\forall n, k \leftarrow Gen(1^n), \forall m\in \{0,1\}^*, Vrfy_k(m,Mac_k(m))=1\]

当 \(Mac\) 是确定性算法时，一种典型的认证方法是重新计算tag并进行比对.

安全性通过消息认证试验 \(Mac-forge_{\mathcal{A},\Pi}(n)\) 来定义： 1. \(k \leftarrow Gen(1^n)\) 对敌手保密. 2. 给 \(\mathcal{A}(1^n)\) 调用 \(Mac_k(\cdot)\) 的能力，\(\mathcal{A}\) 查询若干次，最终输出 \((m,t)\). 记 \(\mathcal{Q}\) 为 \(\mathcal{A}\) 所查询消息的集合. 3. \(\mathcal{A}\) 成功也即 \(Mac-forge_{\mathcal{A},\Pi}(n)=1\) 当且仅当 \(Vrfy_k(m,t)=1 \wedge m \notin \mathcal{Q}\).

一个 MAC 是 secure 当且仅当 对于任何一个PPT的敌手 \(\mathcal{A}\) , 都存在可忽略的函数 \(negl\) ，使得 \[Pr[Mac-forge_{\mathcal{A},\Pi}(n)=1]\leq negl(n).\]

secure 保证了一个敌手不可能为一个先前未认证的新消息生成一个合法的tag , 但是并未保证敌手不能为已经认证的消息生成一个新的合法 tag, 于是有了 strongly secure

消息认证试验 \(Mac-sforge_{\mathcal{A},\Pi}(n)\) : 1. \(k \leftarrow Gen(1^n)\) 对敌手保密. 2. 给 \(\mathcal{A}(1^n)\) 调用 \(Mac_k(\cdot)\) 的能力，\(\mathcal{A}\) 查询若干次，最终输出 \((m,t)\). 记 \(\mathcal{Q}\) 为 \(\mathcal{A}\) 所查消息和对应tag对的集合. 3. \(\mathcal{A}\) 成功也即 \(Mac-sforge_{\mathcal{A},\Pi}(n)=1\) 当且仅当 \(Vrfy_k(m,t)=1 \wedge (m,t) \notin \mathcal{Q}\).

一个 MAC 是 strongly secure 当且仅当 对于任何一个PPT的敌手 \(\mathcal{A}\) , 都存在可忽略的函数 \(negl\) ，使得 \[Pr[Mac-sforge_{\mathcal{A},\Pi}(n)=1]\leq negl(n).\]

显然典型的使用确定性的 secure 的 MAC 天然就是 strongly secure 的.

如何使用 MAC

What if Alice needs to send a message in Q, (i.e. was sent previously)? + Directly applying the MAC would cause repeated messages to be rejected. + One can append a unique timestamp or nonce to each message.

构造 secure 的 MAC

定长 MAC

如果有函数 \(l\) 使得对于任意 \(k \leftarrow Gen(1^n)\), \(Mac_k\) 仅对 \(m\in \{0,1\}^{l(n)}\) 上的消息有定义.

使用PRF！

令 \(F\) 是 PRF . 对长为 n 的消息定义一个定长的 MAC： + Mac : 输入密钥 \(k\in \{0,1\}^n\) 和消息 \(m\in\{0,1\}^n\),输出标签 \(t:=F_k(m)\). + Vrfy: 输入密钥 \(k\in \{0,1\}^n\) 消息 \(m\in \{0,1\}^n\) 和标签 \(t\in\{0,1\}^n\),输出1当且仅当 \(t=F_k(m)\).

若 \(F\) 是 PRF ， 上述构造就是一个对长度为n的消息的 secure fixed-length MAC.

处理长消息（定长）

basic CBC-MAC: \(F\) 是一个 PRF ， 以及长度函数 \(l\) + Mac : 输入 \(k \in \{0,1\}^n, m\in \{0,1\}^{n\cdot l(n)}\) + 将 \(m\) 分成 \(l(n)\) 个长度为 \(n\) 的块, + \(t_0:= 0^n\),对 \(i\) 从 1 到 \(l(n)\), \(t_i := F_k(t_{i-1} \oplus m_i)\) + 输出 \(t_l\) + Vrfy : 输入 \(k\in \{0,1\}^n, m, t\),输出1当且仅当 \(|m|=n\cdot l(n) \wedge t=Mac_k(m)\).

仍然仅能处理定长的消息！！！

AE

消息的安全性和真实性都要！

定义

一个私钥加密方案是 AE 的如果其是 CCA-secure 并且 unforgeable.

CCA安全

CCA安全由CCA indistinguishability experiment \(PrivK_{\mathcal{A},\Pi}^{cca}(n)\) 定义： 1. \(k\leftarrow Gen(1^n)\) 2. 给敌手调用 \(Enc_k(\cdot),Dec_k(\cdot)\) 的权限，敌手输出 \(m_0,m_1\). 3. 加密 \(c\leftarrow Enc_k(m_b),b\overset{\$}{\leftarrow}\{0,1\}\),将 \(c\) 发送给敌手. 4. 敌手调用 \(Enc_k(\cdot),Dec_k(\cdot)\) ，但不允许直接调用 \(Dec_k(c)\) ，输出\(b'\). 5. \(PrivK_{\mathcal{A},\Pi}^{cca}(n)=1\) 当且仅当 \(b=b'\).

私钥加密方案 \(\Pi\) 是 \(CCA-secure\) 的当且仅当对于任意PPT的敌手均有 \[Pr[PrivK_{\mathcal{A},\Pi}^{cca}(n)=1]\leq \frac{1}{2} + negl(n).\]

可延展性（malleable) 在不知道 \(m\) 的前提下构造其密文. CCA安全推出了不可延展性.

不可伪造性

不可伪造加密试验 \(Enc-Forge_{\mathcal{A},\Pi}(n)\): 1. \(k\leftarrow Gen(1^n)\) 2. 敌手有调用 \(Enc_k(\cdot)\) 的权限， 最终输出一个密文. 3. \(m:=Dec_k(c)\) ， \(Enc-Forge_{\mathcal{A},\Pi}(n)=1\) 当且仅当 密文合法且 \(m\) 没被查过.

私钥加密方案 \(\Pi\) 是 unforgeable 当且仅当对于任意PPT的敌手均有 \[Pr[Enc-Forge_{\mathcal{A},\Pi}(n)=1]\leq negl(n).\]

哈希函数

将长串映射到短串！！！

加密哈希函数

能在加密应用中使用的符合密码学要求的哈希函数，一般满足如下要求： + collision resistant: It is difficult to find two different messages \(m\) and \(m′\) such that \(hash(m) = hash(m′)\). + “completely unpredictable” (a.k.a. random oracles): The hash function is indistinguishable from a random mapping.

抗碰撞哈希函数

我们将键控哈希函数 \(H\) 视为一个以密钥 \(s\) 和字符串 \(x\) 为输入的函数 \(H^s(x)=H(s,x)\).

这儿的密钥 \(s\) 通常并不是随机采样出来的，而是由密钥生成算法 \(Gen\) 生成的.此外，密钥 \(s\) 并不保密...

输出长度为 \(l\) 的哈希函数是这么一对 PPT 的算法 (Gen,H), 满足: + \(s \leftarrow Gen(1^n)\) + \(H\) 以密钥 \(s\) 和串 \(x\in\{0,1\}^*\) 为输入，输出串 \(H^s(x) \in \{0,1\}^{l(n)}\).

如果 \(H^s\) 仅对定长 \(l'(n)>l(n)\) 的输入有定义，我们就称之为定长哈希函数，也叫compression function.

碰撞检测试验

The collision-finding experiment \(Hash\text{-}coll_{\mathcal{A},\Pi}(n)\): 1. \(s \leftarrow Gen(1^n)\) 2. 给敌手 \(s\), 敌手输出 \(x,x'\) 3. \(Hash\text{-}coll_{\mathcal{A},\Pi}(n)=1\) 当且仅当 \(H^s(x)=H^s(x')\)

一个哈希函数 \(\Pi=(Gen,H)\) 是抗碰撞（collision resistant）的当且仅当对任意PPT的敌手 \(\mathcal{A}\) ,都有\[Pr[Hash\text{-}coll_{\mathcal{A},\Pi}(n)=1]\leq negl(n).\]

我们有时候省略掉 \(Gen\) ，直接称 \(H\) 或 \(H^s\) 为抗碰撞哈希函数.此外，在实践中，有很多非键控抗碰撞哈希函数.

更弱的抗碰撞 + 2nd p.r. 给一个均匀随机的 \(x\)，找 \(x'\) 使得 \(H^s(x)=H^s(x')\). + p.r. 给一个均匀随机的 \(y\), 找 \(x'\) 使得 \(y=H^s(x')\).

\[ c.r. \Rightarrow 2nd\ p.r. \Rightarrow p.r.\]

Hash and MAC

至此，我们有对定长消息做认证的MAC，也有将任意长度的串映射到定长串的Hash，二者的结合注定发生奇妙的化学反应.

令 \(\Pi=(Mac,Vrfy)\) 是对长度为 \(l(n)\) 的消息的 MAC , \(\Pi_H=(Gen_H,H)\) 是输出为定长 \(l(n)\) 的哈希函数, 构造一个MAC \(\Pi'=(Gen',Mac',Vrfy')\) 适用于任意长度的消息： + Gen': 输入 \(1^n\) ，均匀选择 \(k\in\{0,1\}^n\) , 运行 \(Gen_H(1^n)\) 得到 \(s\) , 密钥 \(k':=<k,s>\). + Mac': 输入密钥 \(<k,s>\) , 和任意长度的消息 \(m\in\{0,1\}^*\) , 输出 \(t \leftarrow Mac_k(H^s(m))\). + Vrfy': 输入密钥 \(<k,s>\) , 消息 \(m\in\{0,1\}^*\) , tag \(t\) , 输出1当且仅当 \(Vrfy_k(H^s(m),t)=1\).

如果 \(MAC\) 是secure的 , 且 \(\Pi_H\) 是抗碰撞的， 那么上述构造是对任意长度的消息的一个secure的MAC.（Pf.两种情况分别规约到c.r. Hash和secure MAC上）

由鸽巢原理，找 \(2^{l+1}\) 个值总能找到碰撞 \(\Theta(2^l)\), 有没有可能少找一些期望有一定概率找到呢？

生日问题： > If q people are in a room, what is the probability that two of them have the same birthday?

若 \(y_1,...,y_q\) 是从 \(\{1,...,N\}\) 中均匀抽取的，当 \(q=\Theta(N^{1/2})\) 时，有接近 1/2 的概率存在碰撞.

The above imply that a hash function with n-bit output is limited to n/2 bits of security.

单向函数

什么是OWF？ + easy to compute + hard to invert

我们给出OWF的形式化定义.一个函数 \(f:\{0,1\}^* \rightarrow \{0,1\}^*\) 是 one-way 如果满足以下两个条件： 1. Easy to compute : 存在PPT的算法 \(M_f\) 来计算 f (多项式时间内可计算) 2. Hard to invert: \(\forall PPT \mathcal{A} , Pr_{x\overset{\$}{\leftarrow}\{0,1\}^n}[\mathcal{A}(1^n,f(x))\in f^{-1}(f(x))]\leq negl(n)\).

注意，此处的 \(x\overset{\$}{\leftarrow}\{0,1\}^n\) 是均匀选取的，并且我们只要求给出原象，不求给出 \(x\) 本身.

我们当然也有inverting experiment \(Invert_{\mathcal{A},f}(n)\): 1. 均匀选取 \(x\in\{0,1\}^n\) , 计算 \(y:=f(x)\) 2. 敌手以 \(y\) 为输入， 输出 \(x'\) 3. \(Invert_{\mathcal{A},f}(n)=1/ iff./ f(x')=y\)

Hard to invert: 对任意PPT的敌手 \(\mathcal{A}\) , \(Pr[Invert_{\mathcal{A},f}(n)=1]\leq negl(n)\).

硬核谓词(Hard-Core Predicates)

\(f(x)\) 是OWF意味着我们无法从 \(f(x)\) 中还原出 \(x\), 但并不意味着 \(f(x)\) 不会泄露 \(x\) 的信息，就比如 \(f(x_1||x_2)=x_1||g(x_2)\).

我们定义函数 \(f\) (不一定是OWF) 的硬核谓词 \(hc:\{0,1\}^*\rightarrow \{0,1\}\) 如果 \(hc\) 可以在多项式时间内计算，并且对任意PPT的 \(\mathcal{A}\) 有 \[Pr_{x\overset{\$}{\leftarrow}\{0,1\}^n}[\mathcal{A}(1^n,f(x))=hc(x)]\leq 1/2 + negl(n)\].

这儿的 \(x\overset{\$}{\leftarrow}\{0,1\}^n\) 依然是均匀选取的.另外，我们没要求函数 \(f\) 一定是OWF.

任意OWF的硬核谓词

Goldreich-Levin Theorem: Assume one-way function (resp. permutation) exists. Then there exists a one-way function (resp. permutation) \(g\) and a hard-core predicate \(hc\) of \(g\).

这里的 \(r\) 是均匀随机选取的.

使用OWF构造PRG

令 \(f\) 是一个one-way permutation，\(hc\) 是 \(f\) 的硬核谓词，那么 \(G(s)=f(s)||hc(s)\) 是一个 PRG 扩展因子为 \(l(n)=n+1\).

下面开始叠叠乐！

如果有一个 扩展因子为 \(n+1\) 的PRG，那么对任意多项式 \(poly\) ，存在PRG \(\hat{G}\) 扩展因子为 \(poly(n)\).

Factoring Assumption

\(GenModulus(1^n)\): + 输出 \((N,p,q)\) + \(N=pq\) + 除可忽略的概率 \(negl(n)\) 外，\(p,q\) 均为 \(n-bit\) 素数.

The factoring experiment \(Factor_{\mathcal{A},GenModulus}(n)\): 1. \((N,p,q) \leftarrow GenModulus(1^n)\) 2. 给 \(\mathcal{A}\) \(N\) , 要其输出 \(p',q'\) 3. \(Factor_{\mathcal{A},GenModulus}(n)=1\) iff. \(N=p'q'\).

Factoring is hard relative to GenModulus if for all PPT algorithms \(\mathcal{A}\) there exists a negligible function \(negl\) such that \[Pr[Factor_{\mathcal{A},GenModulus}(n)=1]\leq negl(n).\]

Factoring Assumption就是假设存在这么一个GenModulus使factoring是hard的.

RSA problem

\(GenRSA(1^n)\) : + 输出 \((N,e,d)\) + \(N\) 是两个 \(n-bit\) 素数的乘积 + \(gcd(e,\phi(N))=1, ed=1\ mod\ \phi(N)\)

The RSA experiment \(RSA-inv_{\mathcal{A},GenRSA}(n)\): 1. \((N,e,d)\leftarrow GenRSA(1^n)\) 2. 均匀选取 \(y\in \mathbb{Z}_N^*\). 3. 给 \(\mathcal{A}\) \((N,e,y)\) , 输出 \(x\in \mathbb{Z}_N^*\) 4. \(RSA-inv_{\mathcal{A},GenRSA}(n)=1\) iff. \(x^e=y\).

The RSA problem is hard relative to GenRSA if for all PPT algorithms \(\mathcal{A}\) there exists a negligible function \(negl\) such that \[Pr[RSA-inv_{\mathcal{A},GenRSA}(n)=1]\leq negl(n).\]

RSA Assumption就是假设存在这么一个GenRSA使RSA problem是hard的.

离散对数问题

对于 \(h\in \mathbb{G}\) , \(\mathbb{G}\) 是一个循环群, 存在唯一的 \(x\in Z_q, q=|\mathbb{G}|\), 使得 \(g^x=h\), 我们称 \(x\) 是 \(h\) 的离散对数 w.r.t \(g\) , 写作 \(x=log_g\ h\).

离散对数问题就是求解一个在循环群上均匀抽取的元素的离散对数，可定义为以下试验.

The discrete-logarithm experiment \(DLog_\{\mathcal{A},\mathcal{G}\}\): 1. \((\mathbb{G},q,g)\leftarrow \mathcal{G}(1^n)\), 其中 \(\mathbb{G}\) 是循环群, \(q=|\mathbb{G}|,\overline{||q||=n}\), \(g\) 是生成元. 2. $h $ 3. 给 \(\mathcal{A}\) 输入 \(\mathbb{G},q,g,h\) , 其输出 \(x\in \mathbb{Z}_q\). 4. \(DLog_\{\mathcal{A},\mathcal{G}\}=1\) iff. \(g^x=h\).

We say that the discrete-logarithm problem is hard relative to \(\mathcal{G}\) if for all PPT algorithms \(\mathcal{A}\) there exists a negligible function \(negl\) such that \[ Pr[DLog_\{\mathcal{A},\mathcal{G}\}=1]\leq negl(n) .\]

离散对数假设就是假定存在这么一个 \(\mathcal{G}\).

D-H Problem

The computational D-H (CDH) problem: Given \(g\), \(g^x\) and \(g^y\), can you compute \(g^{xy}\)?

The decisional D-H (DDH) problem: Given \(g\), \(g^x\), \(g^y\), and g^{xy}, can you differentiate \(g^{xy}\) from a uniform random group element \(g^z\)?

The D-H assumptions are assumptions that there exists instances of CDH/DDH problem which are hard.

Closed related to the discrete-logarithm problem, but not known to be equivalent.
